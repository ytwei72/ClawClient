# 网关重新配对：手机端与 OpenClaw 侧

当云端/OpenClaw **已删除设备** 或 **始终看不到待批准** 时，需要 **两端** 一起排查：客户端清掉所有与会话绑定的状态；服务端清掉已配对记录或待处理请求。

## 一、手机端（灵犀 Claw / 本应用）

应用内会持久化多类与网关相关的数据，仅删 `device.json` 往往不够。

### 自动清理（推荐）

1. **首次进入连接引导**：会执行与下面等价的「配对重置」（并断开连接）。
2. **设置 → 设备 → 「重置网关配对（本机）」**：需确认对话框；适合已完成引导、不从引导页进入的情况。

### `clearAllGatewayPairingState()` 实际会清什么

在 `SecurePrefs.clearAllGatewayPairingState()` 与 `NodeRuntime.clearLocalDeviceIdentityForOnboarding()` 中，**本机**会处理：

| 类型 | 说明 |
|------|------|
| `files/openclaw/identity/device.json` | Ed25519 设备密钥（`deviceId` / 签名来源） |
| 所有 `gateway.deviceToken.*`（加密偏好） | 按 `deviceId`+角色缓存的网关 **设备令牌** |
| `gateway.manual.token`、当前实例的 `gateway.token.*` / `bootstrapToken` / `password` | **安装/共享令牌**、引导令牌、密码 |
| **`node.instanceId`（轮换为新 UUID）** | 连接 JSON 里的 `client.instanceId`；与旧实例绑定的令牌键一并失效 |
| 所有 `gateway.tls.*`（普通偏好） | 已保存的 TLS 指纹（下次连同一主机可能再次提示信任） |
| `gateway.lastDiscoveredStableID` | 上次 mDNS 选中的网关 id |

之后仍会 **断开** operator/node WebSocket。

### 仍看不到「待批准」时

只要连接请求里带上 **有效的网关共享令牌**（`auth.token`），多数网关配置会 **直接认证通过**，**不会**再走待批准列表。此时应：

- 手动连接时 **令牌留空** 再点连接；或  
- 在 OpenClaw 侧调整策略；或  
- 换用仅设备身份的配对流程（以你方网关版本为准）。

## 二、OpenClaw 侧（网关所在机器）

以下内容整理自官方文档：[devices CLI](https://docs.openclaw.ai/cli/devices)（以你安装的 OpenClaw 版本为准）。

### 查看待批准与已配对

```bash
openclaw devices list
openclaw devices list --json
```

### 拒绝一条待批准请求

```bash
openclaw devices reject <requestId>
```

### 批准（含批准最近一条）

```bash
openclaw devices approve <requestId>
openclaw devices approve --latest
```

### 删除已配对设备（按 deviceId）

```bash
openclaw devices remove <deviceId>
```

### 批量清空

```bash
openclaw devices clear --yes
openclaw devices clear --yes --pending
```

### 按角色吊销设备令牌（不断开名单时也可能需要）

```bash
openclaw devices revoke --device <deviceId> --role node
openclaw devices revoke --device <deviceId> --role operator
```

远程网关上执行 CLI 时通常需要显式传入（示例）：

```bash
openclaw devices list --url ws://主机:端口 --token <网关令牌>
```

（具体选项以 `openclaw devices --help` 为准。）

### 配置与缓存目录（供人工排查）

常见用户配置与数据在用户目录下，例如 `~/.openclaw/`（见各环境导出文档）。若服务端仍有脏状态且 CLI 无法清理，需在**备份后**由管理员对照官方文档处理工作区/状态文件——**勿盲目删目录**，以免损坏会话与代理配置。

### 官方「令牌不一致」排查提要

文档中的 Token drift 建议步骤包括：`openclaw config get gateway.auth.token`、`devices list`、对受影响设备 `devices rotate` 或 `remove` 后重新 `approve`。详见上述 devices 文档中的 *Token drift recovery checklist*。

---

## 三、推荐联调顺序

1. 手机：**重置网关配对** 或 **重新走引导**；手动连接时 **令牌留空**（若你要看到 pending）。  
2. 网关：`openclaw devices list` 是否出现新请求。  
3. 若无：`devices clear --yes --pending`（谨慎）或删掉对应条目后再连。  
4. 批准后再按需让用户在应用内填回安装令牌（若你的部署依赖共享令牌）。

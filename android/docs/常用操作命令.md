# ADB 常用操作命令

## 安装 APK

```powershell
adb -s 84157cb0 install .\openclaw-2026.3.24-play-debug.apk
```

### 安装到第一个设备（通常是物理手机）

```powershell
adb -d install .\openclaw-2026.3.24-play-debug.apk
```

### 安装到第一个模拟器

```powershell
adb -e install your_app.apk
```

## Logcat 常用命令

| 命令 | 作用说明 |
| --- | --- |
| `adb logcat -c` | 清空当前 logcat 缓存，不输出日志，只做清理。 |
| `adb logcat *:E` | 实时输出所有 Error 级别的日志，过滤掉其他级别。 |

## 语音唤醒使用步骤（Vosk / HotwordService）

### 1) 启用前检查（模型文件）

请先确认 `android/app/src/main/assets/model` 中至少存在以下文件：

- `am/final.mdl`
- `conf/model.conf`
- `graph/HCLr.fst`
- `graph/Gr.fst`
- `graph/phones/word_boundary.int`

如果缺失，请先补齐模型文件，否则热词服务会启动失败或无法进入监听。

### 2) 安装并启动 App

```powershell
adb -d install .\openclaw-2026.3.24-play-debug.apk
```

安装完成后打开 App。

### 3) 在 App 内配置语音唤醒

1. 打开「设置」页面。
2. 在「媒体」中授予「麦克风」权限（`RECORD_AUDIO`）。
3. 进入「语音唤醒」分组，选择模式：
   - 关闭
   - 仅前台（App 在前台时监听）
   - 始终监听（前后台都监听，更耗电）
4. 在「唤醒词（逗号分隔）」中输入关键词（例如：`openclaw, claude`），点击「保存」。
5. 点击「测试唤醒」进行链路自检。

### 4) 验证是否生效

- 观察通知栏是否出现“OpenClaw 语音唤醒”前台服务通知。
- 在设置页「热词调试日志」中确认有“热词服务启动 / 开始热词识别 / 命中唤醒词”等记录。

### 5) ADB 调试命令（排错）

```powershell
# 清空日志缓存
adb logcat -c

# 只看与热词相关关键字日志（PowerShell 可直接使用 findstr）
adb logcat | findstr /I "HotwordService HotwordDebugLogger vosk wake"
```

### 6) 常见问题

- 缺少麦克风权限：会出现“缺少 RECORD_AUDIO 权限”并停止服务。
- 模型不完整：会出现“模型目录为空 / 模型修复失败 / 未找到可用模型”。
- 麦克风被占用：正在语音会话时，热词监听会暂停，需结束会话后再测。

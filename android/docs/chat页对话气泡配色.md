好的，我来设计一套完整的手机 chat 界面配色与气泡方案。以下是完整的配色逻辑说明：

**颜色选择原则：角色 → 色系，partType → 形态**

每种 role 绑定一个专属色调，partType 只改变气泡的**形态**（实心/虚线/深色代码块），颜色不变。这样用户一眼就能辨认当前是谁在说话、在做什么。

| Role | 色系 | 选用原因 |
|---|---|---|
| user | 紫色（#EEEDFE / #534AB7） | 主动发起，紫色有"意图"感，靠右对齐 |
| assistant · text | 青绿色（#E1F5EE / #0F6E56） | 自然、可信，回复类内容的经典色 |
| assistant · thinking | 灰色虚线边框 + 斜体 | 视觉上"淡出"，暗示这是中间过程不是最终结论 |
| assistant · toolCall | 深色代码底（#2C2C2A） | 工具调用是 JSON 代码，延续代码编辑器的暗色惯例，与文字气泡形成强区分 |
| toolResult | 琥珀色（#FAEEDA / #EF9F27） | 工具结果是"外部数据回传"，琥珀偏黄暖，与 toolCall 的深色形成呼应 |

**细节设计要点：**

- **气泡尾角**：user 右下圆角收小（4px），assistant/toolResult 左下收小，符合主流 IM 习惯
- **toolCall 内语法高亮**：JSON key 用亮黄，string 用青绿，数字用浅橙，boolean 用粉色，利于快速扫读
- **thinking 带动态省略点**：三个圆点 bounce 动画，视觉上传达"正在进行"
- **role-label 微标签**：每组气泡上方有极小的分类标签（10px 大写），方便调试期快速定位来源，上线后可隐藏
- **深浅模式**：文字均使用 CSS 变量，深色模式自动适配；代码块使用固定深色，不随模式反转

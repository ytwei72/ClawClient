# 项目数据结构规范

本文档记录 Android Chat 侧与 Gateway 会话相关的数据结构约定，重点说明 `session` / `history` / `message.content` 的结构与解析策略。

## 1. Chat 核心模型（Android）

来源文件：`android/app/src/main/java/ai/openclaw/app/chat/ChatModels.kt`

- `ChatHistory`
  - `sessionKey: String`
  - `sessionId: String?`
  - `thinkingLevel: String?`
  - `messages: List<ChatMessage>`

- `ChatMessage`
  - `id: String`（本地生成，用于 UI 稳定渲染）
  - `role: String`（常见：`user` / `assistant` / `system`）
  - `content: List<ChatMessageContent>`
  - `timestampMs: Long?`

- `ChatMessageContent`
  - `type: String`（内容块类型）
  - `text: String?`（文本类内容）
  - `mimeType: String?`（媒体类型）
  - `fileName: String?`
  - `base64: String?`（二进制载荷）

## 2. Gateway `chat.history` 返回结构（约定）

Android 端在 `ChatController.parseHistory()` 解析如下结构：

```json
{
  "sessionId": "string",
  "thinkingLevel": "off|low|medium|high",
  "messages": [
    {
      "role": "user|assistant|system|...",
      "timestamp": 1742970000000,
      "content": [
        {
          "type": "text|thinking|tool_call|tool_result|image|...",
          "...": "..."
        }
      ]
    }
  ]
}
```

## 3. `content` block 解析规则（当前实现）

来源文件：`android/app/src/main/java/ai/openclaw/app/chat/ChatController.kt`，函数 `parseMessageContent()`

- `type == "text"`
  - 优先读取 `text`

- `type in ("thinking", "reasoning", "thought")`
  - 优先读取 `thinking`
  - 兜底读取 `text`

- 其他未知类型
  - 依次尝试 `text` / `thinking`
  - 若 `mimeType == null`，可将 `content` 视为文本
  - 再兜底 `obj.toString()`，确保内容可见

- `base64`
  - 仅在 `mimeType != null` 且 `content` 非空时映射到 `base64`

> 说明：该策略用于兼容不同 provider / 运行时返回的异构内容块，目标是“不丢内容、可调试、可展示”。

## 4. 会话键（Session Key）规范

常见形式：

- `main`（别名）
- `agent:<agentId>:main`（智能体主会话）
- 其他渠道/线程键（如 channel/group/thread 维度）

UI 层在 Chat 页面支持：

- 先选择 agent（映射到 `agent:<id>:main`）
- 再选择该 agent 下的历史会话

## 5. 日志规范（OpenClawChat）

统一日志 tag：`OpenClawChat`

关键日志包含：

- 发送侧：`send userInput=...`、`send paramsJson=...`
- 会话加载：`bootstrap history loaded ...`
- 历史刷新：`refresh history loaded ...`

为便于快速定位，历史日志输出会同时包含：

- `historyRaw`：原始 JSON 截断片段
- `parsed`：解析后的结构化摘要（`role + content[type + 文本片段]`）

示例（伪）：

```text
bootstrap history loaded ... historyRaw=... parsed={role=user, content=[{type=text, text=...}]} | {role=assistant, content=[{type=thinking, text=...}, {type=text, text=...}]}
```

## 6. 展示层规范（Chat UI）

- 用户 `role=user` 文本支持“净化显示 / 原文调试切换”
- `assistant` 按 `type` 使用不同卡片（text/thinking/tool/image/unknown）
- 长文本默认折叠，支持“展开/收起”

> 注意：展示层净化仅影响 UI，不修改原始会话数据。
